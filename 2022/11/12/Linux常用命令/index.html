    系统级命令

    uname

    查看当前系统内核

    • a 查看更多

    tty

    查看当前终端

    uptime

    查看系统的负载情况

    • 可以使用“watch -n 1 uptime”来每秒刷新一次获得当前的系统负载情况

    free

    查看内存的使用情况

    • m以m为单位显示当前系统中内存的使用量情况

    who

    查看当前的登录用户

    last

    查看用户的登录记录

    history

    显示历史输入的命令

    • c清空记录

    文件管理

    系统文件

    /etc/profile //设置系统环境变量

    /etc/bashrc //影响bash bash环境

    /var/log/messages //系统主日志文件

    /var/log/secure //跟安全相关的日志如ssh登录，本地登录…

    touch

    创建空白文件与修改文件时间

    • a近修改“访问时间”（atime）

    • m近修改“更改时间”（mtime）

    • d同时修改atime与mtime

    • t要修改成的时间[YYMMDDhhmm]

      示例

      touch -d “2 days ago” test 将访问与修改时间修改为了2天前

    mkdir

    创建空白的文件夹

    • m默认的文件目录权限，如“-m 755”

    • p连续创建多层目录，递归（若文件夹与存在则忽略）

    • v显示创建的过程

    cp

    复制文件或目录

    • p保留原始文件的属性

    • d若对象为“链接文件”，则保留该“链接文件”的属性

    • r递归持续复制（用于目录）

    • i若目标文件存在则询问是否覆盖

    • f覆盖已存在的文件前不询问

    • v显示操作过程

    • a相当于-pdr

      示例

      当你复制一个路径特别长的文件时，可以这样复制

      cp /etc/path1/path2/path3/{file1,file2}cp /etc/path1/path2/path3/file{1,2}cp /etc/path1/path2/path3/file{,2}

    mv

    剪贴/重命名 文件或路径

    • b: 当目标文件或目录存在时，在执行覆盖前，会为其创建一个备份。

    • f: 如果指定移动的源目录或文件与目标的目录或文件同名，不会询问，直接覆盖旧文件。

    • n: 不要覆盖任何已存在的文件或目录。

    • u：当源文件比目标文件新或者目标文件不存在时，才执行移动操作。

    rm

    删除目录或文件

    • r递归，用于删除目录

    • f删除时不询问是否删除

    • v显示详细过程

    使用rm -rf *时删除的文件是不包含隐藏文件的！

    在Redhat6版本开始，增加了对/目录的保护措施，如果用户使用rm -rf /时会警告用户不能删除/。除非增加参数–no-preserve-root

    cat

    查看纯文本文件

    • n显示行号

    • b显示行号（不对空白行进行编号）

    • s 当遇到有连续两行以上的空白行，就代换为一行的空白行

    • A显示出”不可见“的符号，如空格，tab键等等

    • T将TAB字符显示为^|

    • E在每行结束处显示$

    示例：

    把 textfile1 的文档内容加上行号后输入 textfile2 这个文档里：

    cat -n textfile1 > textfile2

    把 textfile1 和 textfile2 的文档内容加上行号（空白行不加）之后将内容附加到 textfile3 文档里：

    cat -b textfile1 textfile2 >> textfile3

    清空 /etc/test.txt 文档内容：

    cat /dev/null > /etc/test.txt

    cat 也可以用来制作镜像文件。例如要制作软盘的镜像文件，将软盘放好后输入：

    cat /dev/fd0 > OUTFILE

    相反的，如果想把 image file 写到软盘，输入：

    cat IMG_FILE > /dev/fd0

    补充

    在Linux中创建的文件，换行符为“$”占一个字节

    在Windows中创建的文件，换行符为“^M$”占两个字节

    所以在Windows中使用记事本编辑过的Linux脚本就无法在Linux中执行了

    如果想要让脚本执行需要使用yum安装dos2unix，然后执行 dos2unix 文件

    find

    • find [查找路经] 寻找条件 操作

    • 参数

      • name 匹配名称

      • user 匹配所有者

      • –type b/d/c/p/l/f 匹配文件类型（块设备、目录、字符设备、管道、链接文件、文件）

      • prune 忽略某个目录

      • –exec {}; 将搜索到的结果进一步处理

        • exec 参数后面跟的是command命令，它的终止是以;为结束标志的，所以这句命令后面的分号是不可缺少的，考虑到各个系统中分号会有不同的意义，所以前面加反斜杠。

          {} 花括号代表前面find查找出来的文件名。

        示例：

        找出用户linuxprobe的文件并复制到/root/findresults目录

        find / -user linuxprobe –exec cp -arf {} /root/findresults/;

    dd

    用指定大小的块拷贝一个文件，并在拷贝的同时进行指定的转换。

    1. if=文件名：输入文件名，缺省为标准输入。即指定源文件。< if=input file >

    2. of=文件名：输出文件名，缺省为标准输出。即指定目的文件。< of=output file >

    3. ibs=bytes：一次读入bytes个字节，即指定一个块大小为bytes个字节。 obs=bytes：一次输出bytes个字节，即指定一个块大小为bytes个字节。 bs=bytes：同时设置读入/输出的块大小为bytes个字节。

    4. cbs=bytes：一次转换bytes个字节，即指定转换缓冲区大小。

    5. skip=blocks：从输入文件开头跳过blocks个块后再开始复制。

    6. seek=blocks：从输出文件开头跳过blocks个块后再开始复制。 注意：通常只用当输出文件是磁盘或磁带时才有效，即备份到磁盘或磁带时才有效。

    7. count=blocks：仅拷贝blocks个块，块大小等于ibs指定的字节数。

    8. conv=conversion：用指定的参数转换文件。

    示例：

    创造一个560MB的空文件

    dd if=/dev/zero of=file1 bs=560M count=1

    ln

    创建链接文件

    • s 创建软连接

    • v 显示详细的处理过程

    默认不加参数即创建硬链接

    head

    查看纯文本文件的前N行

    • n 10 显示10行

    • n -10 正常输出（如cat），但不显示最后的10行

    tail

    查看纯文本文件的后N行

    • n 10 显示后面的10行

    • f持续刷新显示的内容 非常重要

    tailf

    等同于tail -f

    less

    可以分页显示文件

    按↓向下翻页，可以按↑向上翻页

    more

    可以分页显示文件

    按回车向下翻页，无法往上翻页

    grep

    • b把可执行文件当作文本文件进行搜索

    • c仅显示找到的次数

    • i忽略大小写

    • n显示行号

    • v反向选择

    • 星号（*）在正则表达式表示匹配前一个字符零次或多次

    示例

    想要搜索/etc/passwd中以“root”开头的行

    grep '^root' /etc/passwd

    想要搜索/etc/passwd中以“bash”结尾的行

    grep 'bash$' /etc/passwd

    从安全日志中查看非法登录主机的用户

    grep 'failure' /var/log/secure

    file

    详细查看文件类型

    例如文本文件、二进制文件、管道文件、链接文件、设备文件

    内容 意思
    executable 可执行文件
    block special 块设备
    character special 字符设备
    symbolic link 符号链接（快捷方式）
    named pipe 命名管道

    type

    查看命令类型，查看是内置命令还是某个文件

    • a 查看参数的所有类型

    示例

    查看ll的type

    [root@localhost ~]# type llll is aliased to `ls -l --color=auto'

    查看cd的type

    [root@localhost ~]# type cdcd is a bash builtin           //系统内置命令

    查看for的type

    [root@localhost ~]# type forfor is a bash keyword          //关键字

    查看useradd的type

    [root@localhost ~]# type useradduseradd is /usr/sbin/useradd

    查看ls的所有类型

    [root@localhost ~]# type lsls is aliased to `ls --color=auto'[root@localhost ~]# type -a lsls is aliased to `ls --color=auto'ls is /usr/bin/ls

    stat

    查看文件的属性

    例如文件的名称、大小、权限、atime、mtime、ctime

    Linux文件有四种时间：

    stat anaconda-ks.cfg //查看文件的详细属性(其中包括文件时间属性)

    访问时间(Access)：atime，查看内容

    修改时间(Modify)：mtime，修改内容

    改变时间(Change)：ctime，文件属性，比如权限

    删除时间：dtime，文件被删除的时间

    使用ls -l显示的时间是最后被修改的时间

    拓展示例

    查找在5日之内修改的文件

    find / -mtime -5

    查找已经5日都没有修改过的文件

    find / -mtime +5

    which

    查找命令的绝对路径

    示例

    [root@localhost ~]# which useradd/usr/sbin/useradd

    du

    用于显示目录或文件的大小

    • s或–summarize 仅显示总计。

    • h或–human-readable 以K，M，G为单位，提高信息的可读性。

    用户管理

    useradd的配置文件，即创建用户时使用的配置文件/etc/login.defs

    useradd

    • D显示默认的用户配置

    • d指定用户的家目录

    • e账号有效截至日期，格式YYYY-MM-DD

    • g指定一个初始用户组（必须已存在）

    • G指定一个或多个扩展用户组

    • s指定默认的bash

    • u指定用户的UID

    passwd

    • d 使用户无密码,但是ssh无法使用该用户远程连接

    • l锁定用户禁止其登录

    • u解除锁定

    • S 显示用户的密码状态

    • e用户下次登陆时强制修改密码

    • f强制

    • n指定口令的最短存活期

    • x指定口令的最长存活期

    • w口令要到期前提前警告的天数

    • i口令到期后多少天停用账户。

    userdel

    • f强制删除用户相关文件

    • r不是强制删除

    usermod

    • c填写账号的备注信息

    • usermod -d 新的路径 -m 用户 重新指定用户的家目录并自动旧的数据转移过去

    • e 账号到期时间

    • g 变更所属用户组

    • G 变更扩展用户组

    • L锁定用户禁止其登录系统

    • U解锁

    • s变更默认终端

    • u修改用户的UID

    权限管理

    umask

    umask命令是用来管理文件权限的

    它的使用方法非常的简单，可以键入umask查看现在umask的参数

    可以使用umask 777 来临时修改umask值

    当umask值为777时，用户创造的文件权限为000.

    当umask值为022时，用户创造的文件权限为644，目录权限为755

    因为Linux下你创造的文件都是没有执行权限的，所以当umask值最小的时候，文件权限也还是666

    综上，对于目录而言，777减去umask的值，就是你创建的目录的权限

    对于文件而言，666减去umask的值，就是你创建文件的权限

    基本权限UGO

    chmod

    用于更改文件或目录的权限

    常用的可以使用字母进行权限的更改

    chmod u+x file1chmod u=rwx file1chmod g+x file2chmod u=rwx,g=rw,o=rx file3

    也可以使用数字进行权限的更改

    chmod 777 file1         //给予file1全部的权限chmod 750 file2         //给予file2用户全部权限，所属组拥有读和执行，其他人没有权限

    chown

    更改所有者

    chgrp

    更改所有组

    chattr

    用于改变文件属性

    这项指令可改变存放在ext2文件系统上的文件或目录属性，这些属性共有以下8种模式：

    • a：让文件或目录只可以附加内容。

    • b：不更新文件或目录的最后存取时间。

    • c：将文件或目录压缩后存放。

    • d：将文件或目录排除在倾倒操作之外。

    • i：不得任意更动文件或目录。

    • s：保密性删除文件或目录。

    • S：即时更新文件或目录。

    • u：预防意外删除。

    • R 递归处理，将指定目录下的所有文件及子目录一并处理。

    • V 显示指令执行过程。

    示例

    用chattr命令防止系统中某个关键文件被修改：

    chattr +i /etc/resolv.conf
lsattr /etc/resolv.conf

    会显示如下属性

    ----i-------- /etc/resolv.conf

    让某个文件只能往里面追加数据，但不能删除，适用于各种日志文件：

    chattr +a /var/log/messages

    pwd

    显示当前路径

    • P显示真实路径（即非快捷链接的地址）

    ls

    查看目录中的文件

    • d + 目录 仅查看目录本身

    • h易读的文件容量（如k,m,g）

    • t将文件依修改时间之先后次序列出

    • r将文件以相反次序显示

    • S按文件大小排序

    • A同-a，但不列出目前目录及父目录

    • F在列出的文件名称后加符号，例如可执行文件加”*“，目录则加”/“

    • R若目录下有文件，则列出目录即目录下的文件（依序列出）

    cd

    切换路径

    • 切换到上一次的目录，如“cd -”

    • ~username切换到其他用户的家目录，如“cd ~root”

    more

    查看纯文本文件

    • 数字 预先显示的行数（默认为一页）

    • d显示提示语句与报错信息

    od

    查看特殊格式的文件

    参数 作用
    -t a 默认字符
    -t c ASCII字符
    -t o 八进制
    -t d 十进制
    -t x 十六进制
    -t f 浮点数

    示例：

    创建 tmp 文件：

    $ echo abcdef g > tmp
$ cat tmp
abcdef g

    使用 od 命令：

    $ od -b tmp
0000000 141 142 143 144 145 146 040 147 012
0000011

    使用单字节八进制解释进行输出，注意左侧的默认地址格式为八字节:

    $ od -c tmp
0000000   a   b   c   d   e   f       g  \n
0000011

    使用ASCII码进行输出，注意其中包括转义字符

    $ od -t d1 tmp
0000000   97   98   99  100  101  102   32  103   10
0000011

    使用单字节十进制进行解释

    $ od -A d -c tmp
0000000   a   b   c   d   e   f       g  \n
0000009

    tr

    转换文本文件中的字符

    • tr [原始字符] [目标字符]

    • 将tr.txt文件中的内容转换成大写

      cat tr.txt | tr [a-z] [A-Z]

    wc

    统计指定文本的行数

    • l只显示行数

    • w只显示单词数

    • c只显示字节数

    cut

    通过列来提取文本字符

    • d分隔符 指定分隔符，默认为Tab

    • f指定显示列数

    • c单位改为字符

    • cut -d: -f1 /etc/passwd -d以“:”来做分隔符，-f参数表示只看第一列的内容

    • grep ^root /etc/passwd | cut -d: -f 7 获取root用户的默认bash解释器

    diff

    对比多个文本文件的差异

    • b忽略空格引起的差异

    • B忽略空行引起的差异

    • –brief或-q 仅报告是否存在差异

    • c使用上下文输出格式

    tar

    • tar -czvf 打包 格式.tar.gz

    • tar -zxvf 解压 格式.tar.gz

    • j 是以bzip2的格式进行压缩或解压

    • p保留原始的权限与属性

    • P使用绝对路径来压缩

    • C指定解压到的目录

    • t查看压缩包内有哪些文件

    示例：

    使用绝对路径压缩/etc，此处需注意参数P在参数f前面，否则会报错

    tar -czvPf etc.gz /etc

    将文件解压到指定文件夹中

    tar -zxvf etc.gz -C /tmp

    cal

    显示日历
